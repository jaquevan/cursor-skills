# Story Arcs for Presentations

Adapted from [red-hat-quick-deck](https://github.com/toddward/red-hat-quick-deck)
for Google Slides workflows.

A deck without a narrative arc is just a list of facts. Choose an arc before
writing any content. The arc determines slide order, headline tone, and which
layouts to use.

---

## Arc 1: Journey

**Structure:** Where I started -> What I did -> What I learned -> Where I'm going

**Best for:** Updates, retrospectives, internship reports, sprint reviews

**Slide sequence:**
1. Title: state the journey's timeframe and scope
2. Section opener: preview the key stops on the journey
3. Starting point: where things were at the beginning
4. What happened: the work, the builds, the discoveries
5. Results: what the work produced (metrics, outputs)
6. What's next: where the journey goes from here
7. Closing

**Headline style:** Past tense for what happened, present tense for results,
future tense for what's next.

**Example headlines (journey):**
- "Week 2: from zero to 10 skills"
- "Started with Cursor, built a full productivity workspace"
- "Eval harness proved skill value: 100% vs 55%"
- "Three project areas to pursue next"

## Arc 2: Problem -> Tension -> Resolution

**Structure:** Things are broken -> Here's why it matters -> Here's how to fix it

**Best for:** Proposals, tool introductions, process improvements, pitches

**Slide sequence:**
1. Title: name the problem domain
2. The problem: what's broken, painful, or slow
3. Tension: why this matters, what happens if we don't fix it
4. The solution: what you're proposing
5. How it works: architecture, process, or demo
6. Evidence: results, data, testimonials
7. Call to action: what to do next
8. Closing

**Headline style:** Problem slides use tension words ("broken", "3 weeks",
"manual"). Solution slides use confidence words ("automated", "100%",
"minutes not hours").

**Example headlines (problem-resolution):**
- "Deployment takes 3 weeks minimum"
- "Every week of delay costs $50K in lost productivity"
- "Automated pipeline cuts deployment to 2 days"
- "Pilot proved 85% time reduction in first sprint"

## Arc 3: Myth-Busting

**Structure:** People think X -> Actually Y -> Here's the proof

**Best for:** Research findings, UX insights, challenging assumptions,
competitive analysis

**Slide sequence:**
1. Title: state the conventional wisdom
2. The myth: what people believe
3. The reality: what the data shows
4. Evidence: research, metrics, user quotes
5. Implications: what this means for the team/product
6. Recommendation: what to do differently
7. Closing

**Headline style:** Myths as questions or common phrases. Reality as
direct contradictions. Evidence as specific numbers.

**Example headlines (myth-busting):**
- "Do users actually read the documentation?"
- "73% skip docs entirely and go to Stack Overflow"
- "In-context help reduced support tickets by 40%"

## Headlines Are the Skeleton

Write all headlines BEFORE writing body text. Read them in sequence — they
should tell the complete story on their own. If the headlines read like a
table of contents ("Introduction", "Background", "Findings"), rewrite them
as assertions.

| Label (weak) | Assertion (strong) |
|---|---|
| What I Built | I built 10 skills in 2 weeks |
| Key Highlights | 100% eval pass rate across 11 assertions |
| How I Evaluated | Structured evals proved the skill adds value |
| Where I Can Contribute | Three ADLC projects need UX skill integration |
| Agenda | What I'll walk you through today |
| Thank you | Thank you |

The closing slide is the one exception — "Thank you" is fine as a label.

## Mapping Note Sections to Arc Beats

| Note section | Journey beat | Problem-Resolution beat | Myth-Busting beat |
|---|---|---|---|
| Summary | Section opener | The problem | The myth |
| Key Points | What happened | How it works | The reality |
| Decisions | Results | Evidence | Evidence |
| Open Questions | What's next | Tension points | Implications |
| Next Actions | What's next | Call to action | Recommendation |

---

## Layout Variety by Arc Beat

Don't use the same layout for every beat. Vary based on the emotional weight:

| Beat type | Recommended layouts |
|---|---|
| Opening/setup | Interior title left, Interior large text |
| Evidence/data | Interior three column, Data callouts |
| Details/process | Interior title and body, Two column |
| Impact/results | Interior large text, Data callouts |
| Forward-looking | Interior title left, Interior overview |
| Transition | Divider (simple) |
