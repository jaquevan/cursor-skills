# Tag Scanner

Reads frontmatter `tags` fields across the entire notes repo, builds a shared
tag index, and wires notes to each other via a `## Related Notes` section.

## Workflow

### Step 1: Scan all notes

Walk every `.md` file under `~/Projects/notes/` (excluding `_site/`, `assets/`).
For each file, extract the YAML frontmatter `tags` array and record:

```
{ path, title, date, categories, tags }
```

Build an inverted index: `tag → [list of note paths that carry it]`.

### Step 2: Determine scope

The user can trigger two modes:

| Mode | Trigger phrase | What it does |
|---|---|---|
| **Full index** | "build my index", "scan my notes" | Regenerates `notes/index.qmd` and updates every note |
| **Single note** | "update tags for this note", or after notetaking saves a file | Updates only the target note's Related Notes section |
| **Query** | "what notes are about [topic]" | Returns a list — no files modified |

### Step 3: Inject Related Notes section

For each note being updated, compute related notes by:
1. Finding all notes that share at least one tag
2. Sorting by number of shared tags (descending), then by date (newest first)
3. Taking the top 5

Append or replace the `## Related Notes` section at the end of the file:

```markdown
---

## Related Notes

| Note | Shared Tags | Date |
|------|-------------|------|
| [Title](relative/path.md) | `tag1`, `tag2` | YYYY-MM-DD |
| [Title](relative/path.md) | `tag1` | YYYY-MM-DD |
```

Use relative paths from the note's own location.

If a note already has a `## Related Notes` section, replace it entirely.

### Step 4: Regenerate the tag index (full-index mode only)

Write `~/Projects/notes/tag-index.md`:

```markdown
---
title: "Tag Index"
date: <today>
---

# Tag Index

*Auto-generated by tag-scanner. Last updated: <date>.*

## Tags

### `internship`
- [Note Title](path/to/note.md) — YYYY-MM-DD
- [Another Note](path/to/note.md) — YYYY-MM-DD

### `cursor-skills`
- ...
```

Tags are sorted alphabetically. Notes under each tag are sorted newest first.

### Step 5: Commit the changes

```bash
cd ~/Projects/notes
git add -A
git commit -m "chore: update tag index and related notes"
git push
```

Git must be configured before this works. See `SETUP.md` in the cursor-skills
repo if `git config user.email` returns nothing.

## Composing with Notetaking

After the `notetaking` skill saves a new note, you can chain directly into
tag-scanner by saying "and update related notes" or "link this note". The
tag-scanner will update only the new note and any notes that share its tags.

---

## Notes on Tag Conventions

Tags should follow these conventions (enforced by the notetaking skill's
style-guide, but worth knowing here too):

- Lowercase, hyphen-separated: `cursor-skills`, `red-hat`, `mcp`
- Prefer specific over broad: `quarto` not `tools`
- Reuse existing tags — the more overlap, the better the related-notes links
- People's names as tags enable filtering by person: `zack`, `priya`
